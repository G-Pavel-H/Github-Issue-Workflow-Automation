# Plan — Issue #19: Add formatDuration(ms) helper and apply to cost summary

> Generated by Tsukinome. The technical blueprint for the work, approved before any code.

## Summary

Add a small self-contained TypeScript utility module `src/util/format-duration.ts` exporting `formatDuration(ms: number): string`, following the repo's existing pure-formatter idiom (e.g. `formatUsd` in `src/llm/pricing.ts`). Provide a companion Vitest unit test. Wire the helper into the cost summary so the run duration renders via `formatDuration` instead of a raw millisecond value.

## Approach

Model the new helper on the existing `formatUsd` pattern: a pure function with an explanatory JSDoc comment, no external deps, ESM `.js` import extensions per repo convention. Implement guard-first logic: if input is not finite or negative, return '0ms'. Round to whole milliseconds with `Math.round` before scale selection (so 999.6 → 1000 → '1s'). Then select scale by boundaries: <1000 → `${ms}ms`; <60000 → `${Math.floor(ms/1000)}s`; <3600000 → `${min}m ${sec}s` using floored components; else `${hr}h ${min}m` dropping seconds. Seconds/minutes remainders use `Math.floor` per R11/R12. Place the file under a new `src/util/` directory (no existing util dir shown; smallest sensible home for a cross-cutting helper) with test at `test/util/format-duration.test.ts` mirroring the `describe/it/expect` style of `test/llm/pricing.test.ts`. For the cost summary, the current `renderCostSummary(calls)` in `src/pipeline/cost.ts` does not itself display a duration; if a duration is present in the summary path it must be rendered through `formatDuration`. Since retrieved context shows no duration field in `renderCostSummary`, the wiring is limited to the location where the run's elapsed duration is composed into the cost summary — pass the raw ms through `formatDuration` there. Keep the change minimal and additive; do not alter how durations are measured or stored.

## Affected Files

- `src/util/format-duration.ts` **(add)** — New self-contained utility module exporting formatDuration(ms) with the scale-based formatting rules.
- `test/util/format-duration.test.ts` **(add)** — Vitest unit tests covering every acceptance criterion and boundary edge case.
- `src/pipeline/cost.ts` **(modify)** — Render the run duration in the cost summary via formatDuration instead of a raw millisecond number, importing from the new util module.

## Contracts

- src/util/format-duration.ts: export function formatDuration(ms: number): string — returns '0ms' for non-finite or negative input; '<n>ms' below 1s; '<n>s' below 1m; '<m>m <s>s' below 1h; '<h>h <m>m' at 1h and above; rounds to whole ms before scale selection.
- src/pipeline/cost.ts: renderCostSummary continues to accept LlmCall[]; where a run duration (ms) is surfaced in the summary it is passed through formatDuration(ms) rather than emitted raw.
