# Spec — Issue #17: Add formatDuration helper and integrate into cost summary

**Type:** feature

> Generated by Tsukinome. This file is the source of truth for the work; edits here drive the build.

## Summary

Add a self-contained formatDuration(ms) utility that converts a millisecond count into a compact, human-readable string (e.g. "820ms", "5s", "1m 5s", "1h 0m"), with consistent rounding and defined handling for zero, negative, and non-finite input. Integrate this helper into the run cost summary so durations are shown in the readable format instead of raw milliseconds.

## Requirements

Each requirement is tagged with a confidence level (explicit / inferred / assumption / unknown).

- **R1** `[explicit]` Provide a function formatDuration(ms: number): string in a small, self-contained utility module.
- **R2** `[explicit]` For values under one second, output whole milliseconds with the "ms" suffix (e.g. 820 → "820ms").
- **R3** `[explicit]` For values from one second up to under one minute, output seconds with the "s" suffix (e.g. 5000 → "5s").
- **R4** `[explicit]` For values from one minute up to under one hour, output minutes and seconds (e.g. 65000 → "1m 5s").
- **R5** `[explicit]` For values of one hour or more, output hours and minutes and drop the seconds (e.g. 3600000 → "1h 0m").
- **R6** `[explicit]` Sub-second values are rounded to whole milliseconds.
- **R7** `[explicit]` Zero input returns "0ms".
- **R8** `[explicit]` Negative or non-finite input (NaN, Infinity, -Infinity) returns "0ms".
- **R9** `[explicit]` The utility module has accompanying unit tests covering the documented behaviours and edge cases.
- **R10** `[explicit]` The run cost summary display uses formatDuration to render any duration it shows instead of raw milliseconds.
- **R11** `[inferred]` Seconds and minutes components are rounded to whole units (no fractional seconds/minutes in output).
- **R12** `[assumption]` When seconds round up to a full minute (or minutes to a full hour), the output normalizes carrying to the next unit rather than showing e.g. "60s".
- **R13** `[assumption]` At minute scale, seconds are always shown even when zero (e.g. 60000 → "1m 0s"), and at hour scale minutes are always shown even when zero.

## Non-Goals

- Changing how durations are displayed in places other than the run cost summary.
- Adding localization, internationalization, or configurable unit labels.
- Displaying sub-hour-scale seconds when at hour scale.
- Supporting units larger than hours (days, weeks).

## Edge Cases

- Exactly 1000ms → "1s".
- Exactly 60000ms → minute-scale output.
- Exactly 3600000ms → "1h 0m".
- Seconds rounding up to 60 (e.g. 59999ms).
- Fractional milliseconds requiring rounding.
- Zero, negative, NaN, and Infinity input.
- Very large durations spanning many hours.

## Assumptions

- When seconds/minutes round or carry up to a full higher unit, the output normalizes the carry (no "60s" or "60m").
- At minute scale the seconds component is always shown (including "0s"), and at hour scale the minutes component is always shown (including "0m").
- Sub-hour output does not include hours, and each tier only shows the two most significant units as illustrated by the examples.
